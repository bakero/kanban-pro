## Objetivo
Relacionar tarjetas por dependencias y bloqueos.

## Funcionamiento
- Vincular tarjetas como dependencias o bloqueos.
- Navegar entre tarjetas relacionadas.
- Indicadores de resueltas o pendientes.

## Reglas de negocio
- Una tarjeta no puede depender de si misma.

## Requisitos
- El buscador debe listar tarjetas accesibles.

## Escenarios de prueba
1. Crear dependencia y verificar indicador.
2. Resolver dependencia y validar estado.

## Ultima ejecucion
- Preview: pendiente
- Produccion: pendiente
